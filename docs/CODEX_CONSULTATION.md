# ワークフロー設計の相談

know-pilatesプロジェクトで以下のワークフローを設計しました。セッション間のコンテキスト継続性について意見が欲しいです。

## 提案ワークフロー

各Phase完了時:
1. TodoWrite更新（セッション内タスク管理）
2. WORKLOG.md追記（Git管理の恒久的記録）
3. Notion MCP経由で作業ログ記録（外部コンテキスト共有）
4. Git commit & push

## ファイル構成

### WORKLOG.md (Git管理)
- 時系列の作業履歴
- Phase完了時に記録
- フォーマット: 実施内容、成果物、判断・意図、次のアクション

### PROGRESS.md (Git管理)
- 静的な実装状況サマリー
- 機能の完成度、既知の問題
- 頻繁には更新しない

### Notion (MCP連携)
- WORKLOG.mdの内容 + より詳細なコンテキスト
- Phase完了時に自動記録
- 判断の意図、技術的な理由を重点的に
