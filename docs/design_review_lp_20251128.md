# LP (Landing Page) デザインレビュー - know（ノウ）ピラティススタジオ

**レビュー日:** 2025年11月28日
**レビュー対象:** LP・体験予約フォーム (`https://know-pilates-5vuelvu8m-yks-projects-a2d094d9.vercel.app`)
**技術スタック:** Next.js 16 + Tailwind CSS v4
**ターゲットユーザー:** メイン: 50~60代女性 / サブ: 70代女性、近隣病院の看護師

---

## 総評

全体として、ターゲットユーザー（50〜60代女性）を意識した非常に質の高いデザインと丁寧な実装がされており、素晴らしい第一印象です。カラーパレット、フォント選定、基本的なアクセシビリティへの配慮など、土台がしっかりしています。

この上で、さらに信頼性と使いやすさを向上させるための改善点をいくつか提案します。

---

## 1. ターゲット適合性（信頼性と読みやすさの向上）

### 良い点
*   **配色**: ベージュ系（`primary`）とテラコッタ系（`accent`）の配色は、落ち着きと温かみがあり、ターゲットに安心感を与えています。
*   **フォント**: `Noto Sans JP`（本文用）と`Noto Serif JP`（見出し用）を適切に使い分けており、可読性と品格を両立しています。
*   **基本サイズ**: 本文のフォントサイズが `18px` (`1.125rem`)、行間が `1.8` に設定されており、非常に読みやすいです。

### 改善提案
*   **文字色のコントラストを上げる**:
    *   **現状**: `Hero.tsx`のサブコピーや`Pricing.tsx`の説明文に使われている `text-neutral-600` は、背景色に対してややコントラストが低く、視力が弱い方には読みにくい可能性があります。
    *   **提案**: より濃い `text-neutral-700` (`#404040`) や `text-primary-900` (`#5d4d3a`) に変更することで、可読性が向上し、信頼感が増します。
    *   `Hero.tsx`のTrust signal (`初回体験 3,000円 | 50代・60代の方も安心`) も重要な情報のため、同様に視認性を高めることを検討してください。

---

## 2. スマホ最適化とUI/UX（直感的な操作性）

### 良い点
*   **タップ領域**: `Button.tsx`でボタンの最小サイズが`44px`に確保されており（WCAG準拠）、スマホでのタップしやすさが十分に配慮されています。
*   **予約フォーム**: ステップ形式（日付選択、時間選択、情報入力、確認）になっており、一度にユーザーに求める情報量が制限されているため、入力の心理的負担が少ないです。`input`に`type="email"`や`type="tel"`が適切に設定されており、モバイルでの入力時に適切なキーボードが表示されます。

### 改善提案
*   **料金プランのCTAボタンの文言を統一する**:
    *   **現状**: `Pricing.tsx`で、「おすすめ」プランでない方のボタンが「詳細を見る」となっていますが、クリックすると予約フォーム（`#reservation`）に遷移します。これはユーザーの期待と異なる動きをする可能性があります。
    *   **提案**: 全てのプランのボタン文言を「**体験レッスンを予約**」または「**このプランで予約へ**」のように、次のアクションが明確にわかるように統一することを推奨します。

---

## 3. ブランド統一性とコード品質（メンテナンス性の向上）

### 良い点
*   **共通UIコンポーネント**: `src/components/ui`に`Button.tsx`や`Card.tsx`などがまとめられており、再利用性が高く、一貫性のあるデザインの基盤が作られています。
*   **CSS変数による一元管理**: `globals.css`で`@theme`を使ってカラーパレットやフォントサイズが変数として一元管理されており、デザインのメンテナンス性が非常に高いです。

### 改善提案
*   **ヒーローセクションのCTAボタンを共通コンポーネントで実装する**:
    *   **現状**: `Hero.tsx`のCTAボタンが、共通の`Button`コンポーネントを使わずに、`Link`タグに直接スタイルを記述しています。
    *   **提案**: `Button`コンポーネントの`asChild`プロパティを活用することで、`Link`タグとしての機能を維持したまま、共通の`Button`コンポーネントのスタイルを適用できます。これにより、デザインの一貫性が保証され、コードもDRY（Don't Repeat Yourself）になり、今後のメンテナンスが容易になります。

---

## 最初の具体的な修正提案

まずは、上記で提案した中で最も効果が高く、コードの健全性にも寄与する「**ヒーローセクションのCTAボタンを共通コンポーネントで実装する**」リファクタリングから始めることをお勧めします。

これは、デザインの一貫性を強化し、将来的な変更にも対応しやすいコードベースを維持するために重要です。
